◆ Image

using starmathpp::Image = typedef cimg_library::CImg<float>

The template mechanism of the CImg class does not help here, since the required bit depth of the image depends on the file to be loaded. This happens at runtime and not at compile time. Therefore, for the moment, the image type is fixed to float to which is considered to be a compromise between accuracy and memory consumption. This way all possible bit depths (2, 8, 16 and 32 bit) are supported.

IDEA: Two options to potentially improve this are:

Boost any: https://www.boost.org/doc/libs/1_54_0/doc/html/any/s02.html Boost variant: https://www.boost.org/doc/libs/1_54_0/doc/html/variant.html
